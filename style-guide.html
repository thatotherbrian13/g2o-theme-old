Typography

Typography Improvements

  • H1: Weight changed from 400 to 300 (Light), line-height tightened from 1.2 to 1.1
  • H2: Line-height adjusted from 1.2 to 1.15 for better visual balance
  • H3 & H4: Weight increased from 600 to 700 (Bold), line-height tightened from 1.3 to 1.03
  • Split-Text: Strong tags inside .font-serif now properly display as Work Sans Bold

Heading Hierarchy Improvements (SEO & Accessibility)

  • Single H1 per page: Hero banners use <h1>; other major headings now use <h2> with visual styling classes
  • No skipped levels: Fixed h2→h4 jumps in accordions, scrollers, and three-columns to use proper h2→h3 hierarchy
  • Team member names: Changed from <h2> to <h3> (nested under section headings)
  • New utility classes: .headline-hero, .headline-section, .headline-card decouple visual size from semantic level

January 2026 Comprehensive Site Audit

Complete theme audit covering WCAG 2.1 AA accessibility, SEO, and code cleanup:

  • Accessibility - Focus Indicators: Added :focus-visible styles with 2px sky-colored outline for keyboard navigation
  • Accessibility - Skip Navigation: Added skip link to bypass navigation (.sr-only until focused)
  • Accessibility - External Links: Added aria-label warnings for links opening in new windows
  • Accessibility - Form Labels: Added role="region" and aria-label to form containers
  • Accessibility - Carousels: Enabled keyboard navigation in Swiper components
  • SEO - Meta Tags: Custom g2o_meta_tags() function for meta descriptions, Open Graph, and Twitter Cards
  • SEO - Schema: JSON-LD structured data for Organization and Article types
  • SEO - CLS Prevention: Added explicit width/height attributes to images
  • Performance: Deferred render-blocking CSS/JS resources in header
  • Code Cleanup: Removed ~200 lines of dead/commented code, 5 unused walker files, legacy navigation system
  • Navigation: Consolidated duplicate CSS files (nav.css vs navigation-main.css)

Issues with Previous Styles

  • H1: Used weight 400 instead of 300, causing inconsistent Zodiak appearance
  • H1, H2: Line-height too loose (1.2) for large display text
  • H3 & H4: Weight 600 (Semi Bold) lacked visual impact; line-height 1.3 too loose
  • Split-Text: <strong> tags inherited Zodiak Light weight, breaking the dual-font pattern

Root Cause & Fix

The issue was in build/css/zodiak-override.css:

/* PREVIOUS (Broken) */
html body .font-serif * {
  font-weight: inherit !important;
}

/* REPAIRED */
html body .font-serif *:not(strong):not(b) {
  font-weight: inherit !important;
}

html body .font-serif strong,
html body .font-serif b {
  font-family: 'Work Sans', sans-serif !important;
  font-weight: 700 !important;
}
